Difference between a static and dynamic web page. by Minho Kang

In static pages are prebuilt files loaded and doesn't change after loading. Whereas dynamic pages are live, which means they communicate with server (back-end) or other micro services, to fetch certain data and update the page.

The HTML assignment we just did is good example of static page. These static pages may include stylesheets and javascript files. Even if there are event bindings and css animations that may create user interaction. Generally when there are no communication with server or micro services, these pages are considered static. Also contents you view when you open an email is an example of static HTML.

Applications we often use communicates to server to load and send data. These dynamic pages have two distinguishable ways to update loaded pages. There are server-side rendered dynamic pages where server side language such as PHP, C#, Java, Python, Javascript (NodeJS) renders pages. Also there are client-side dynamic pages built with modern javascript frameworks such as Angular, React, Vue and Ember manipulates DOM element with data fetched from APIs, which can be sometimes called as static, because it can be hosted by static hosting services like AWS S3 Bucket.
